Configuración Inicial en Linux

Configuración básica de usuario y opciones específicas para Linux.

Configuración Básica
git config --global user.name "Luis Rafael Prado"
Establece tu nombre de usuario globalmente
Email
git config --global user.email "luisprado119@gmail.com"
Establece tu email globalmente
Editor
git config --global core.editor "nano"
Configura tu editor preferido
Rama Principal
git config --global init.defaultBranch main
Establece 'main' como rama principal

Historial y Diferencias

Comandos para explorar el historial del repositorio y ver diferencias.

Ver Historial
git log
Muestra el historial de commits
Historial Compacto
git log --oneline
Historial en una línea por commit
Ver Diferencias
git diff
Muestra cambios no staged
Diferencias Staged
git diff --staged
Muestra cambios en staging area

Autenticación SSH en Linux

Configuración de claves SSH para autenticación segura.

Generar Clave SSH

ssh-keygen -t ed25519 -C "luisprado119@gmail.com"

Agregar al Agente SSH

eval "$(ssh-agent -s)" ssh-add ~/.ssh/id_ed25519

Verificar Conexión

ssh -T git@github.com

🧠 Quiz Final - Pon a Prueba tus Conocimientos

¿Cuál es el comando correcto para crear una nueva rama llamada "feature" y cambiar a ella inmediatamente?
git branch feature && git checkout feature
git checkout -b feature
git new-branch feature
git create feature
¿Qué comando usas para ver los cambios que están en el staging area pero aún no han sido commiteados?
git diff
git diff --staged
git status --staged
git show staged

📋 Flujo de Trabajo Recomendado

Antes de empezar

git pull

Actualiza tu rama local con los últimos cambios

Crear rama

git checkout -b feature/nueva-funcionalidad

Crea una rama descriptiva para tu nueva funcionalidad

Hacer commits

git add . && git commit -m "mensaje descriptivo"

Commits frecuentes con mensajes claros

Subir cambios

git push -u origin feature/nueva-funcionalidad

Sube tu rama al repositorio remoto

Crear Pull Request

Usa la interfaz web de GitHub/GitLab para crear un PR

Después del merge

git checkout main && git pull && git branch -d feature/nueva-funcionalidad

Limpia tu repositorio local después del merge
